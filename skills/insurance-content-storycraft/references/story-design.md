# Story Design

## Core idea

Do not start with explanation. Start with change.

## Minimal story spine

1. A person is in a stable state.
2. Something disrupts that state.
3. The person wants something.
4. Resistance appears.
5. A turn changes the meaning.
6. End with a clearer judgment or next action.

## Pressure-choice check

A story scene is weak if it only says what happened. It needs pressure and a choice.

Before drafting a case story, identify:

- pressure moment: what made the situation hard right now?
- choice: what did the speaker, client, or other person choose to do?
- constraint: what could not be known, promised, or controlled?
- evidence: what concrete action showed character?

If there is no pressure-choice moment, the piece is probably an explanation, not a story.

## McKee concepts to use

### Inciting incident

Use a concrete trigger:

- 一句话误解
- 一个咨询现场
- 一个理赔节点
- 一个热点新闻

### Antagonistic force

In this domain, conflict often comes from:

- 预算有限
- 侥幸心理
- 家人意见不一
- 信息过载
- 对保险不信任
- 怕被销售

### Value shift

Make sure the piece changes one value state, for example:

- 模糊 -> 清楚
- 拒绝 -> 愿意理解
- 侥幸 -> 警觉
- 被动 -> 主动

For short insurance content, use small value shifts. Do not exaggerate change.

Good scale:

- 乱 -> 知道先处理哪一步
- 绷着 -> 松一点
- 只看产品 -> 开始看流程
- 一个人扛 -> 有人一起拆问题

## Compact content templates

### Case template

1. Open with a surprising judgment
2. Show a real scene
3. Name the hidden conflict
4. State the takeaway
5. End with light CTA

### Pressure-choice template

1. Start from the sentence or moment that created pressure.
2. Show what was uncertain or constrained.
3. Show the first concrete action.
4. Let the audience infer the trait.
5. Keep the takeaway modest.

### Hot-topic template

1. Mention the event
2. Translate it into a common-person problem
3. Connect to insurance/risk logic
4. Offer one next step
