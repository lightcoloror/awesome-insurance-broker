# Daily Evidence And Writeback Rules

- Logseq is the fact baseline for daily journal planning and review.
- Obsidian is an AI-organized reference layer for synthesized guidance, examples, and longer learning outputs.
- Use local Markdown reads only as fallback or to preserve Logseq outline shape.
- Real Logseq writes require explicit human approval of the exact operation.
- Do not overwrite Logseq files directly.
- Do not expose local tokens, bearer headers, cookies, or secret env files.
- Prefer one daily long-term minimum action over multiple attractive but competing tasks.
- If the weekly plan and daily impulse conflict, prefer the weekly plan unless the user explicitly reprioritizes.
